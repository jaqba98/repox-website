Build Project

The build project command in Repox is designed for building TypeScript projects with the `@app/ts` type within your monorepo. This is particularly useful when you need to compile TypeScript source code and perform project-specific build tasks. Follow the guide below to use this command effectively.

Command Syntax

repox build project --name=project_name

Parameters

--name: Required. Specifies the name of the TypeScript project to be built with the `@app/ts` type.

Usage Examples

Basic Usage

repox build project --name=example

This command compiles the TypeScript source code and performs project-specific build tasks for the project named "example" with the `@app/ts` type.

Using Aliases

You can use shortcuts to make the command more concise:

repox b p -n=example

Here, b is an alias for build, p is an alias for project, and -n is an alias for --name.

Additional Information

- The `build project` command is specifically tailored for TypeScript projects with the `@app/ts` type.

- The project build configuration will be created if it does not exist or updated if it already exists.

- Ensure you have backups or version control in place if necessary.
