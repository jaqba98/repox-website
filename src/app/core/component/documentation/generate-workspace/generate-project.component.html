Generate Project

The `generate project` command in Repox enables you to create a new project within a specified workspace of your monorepo. Projects are individual components or applications that contribute to the overall functionality of your workspace. Follow the guide below to effectively utilize this command.

Command Syntax

repox generate project --name=project_name --type=@app/ts [--path=/desired/path]

Parameters

--name (or -n): Required. Specifies the name of the project to be generated.

--type (or -t): Required. Specifies the type of the project. Allowed values are @app/ts, @lib/ts, @tool/ts.

--path (or -p): Optional. Specifies the path to the destination where the project will be created.

Usage Examples

Basic Usage

repox generate project --name=project_name --type=@app/ts

This command creates a new project named "project_name" of type "@app/ts" within the specified workspace.

Using Aliases

You can use aliases to make the command more concise:

repox g p -n=project_name -t=@app/ts

Here, g is an alias for generate, p is an alias for project, -n is an alias for --name, and -t is an alias for --type.

With Custom Path

repox g p -n=project_name -t=@app/ts --path=/desired/path

This command creates a new project with the specified name and type in the provided custom path.

Command Options

--name (or -n): Specifies the name of the project. This parameter is required.

--type (or -t): Specifies the type of the project. This parameter is required.

--path (or -p): Specifies the path to the destination where the project will be created. This parameter is optional.

Additional Information

- Ensure that the project name is unique within the specified workspace.

- Supported project types are @app/ts for applications, @lib/ts for libraries, and @tool/ts for tools.

- Aliases (-n, --name, -t, --type, g, p, -p, --path) provide a shorter and more convenient way to use the command.
