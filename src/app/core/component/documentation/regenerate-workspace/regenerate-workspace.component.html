Regenerate Workspace

The regenerate workspace command in Repox allows you to update an existing workspace within your monorepo. This is particularly useful when you need to apply changes to the workspace configuration or structure. Follow the guide below to use this command effectively.

Command Syntax

repox regenerate workspace --force

Parameters

--force: Required. Indicates that the regeneration should proceed, overwriting any existing configuration.

Usage Examples

Basic Usage

repox regenerate workspace --force

This command regenerates the existing workspace, applying any updates to the configuration or structure. The --force option is required to initiate the regeneration.

Using Aliases

You can use aliases to make the command more concise:

repox r w -f

Here, r is an alias for regenerate, w is an alias for workspace, and -f is an alias for --force.

Additional Information

- The regeneration process may overwrite existing workspace configuration. Ensure you have backups or version control in place if necessary.

- The --force option is mandatory to confirm your intention to regenerate the workspace.
